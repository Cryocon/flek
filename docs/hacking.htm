Flek Style Guide

Flek adopts a minimalist C++ philosophy. Flek is written in a subset of the ISO C++ standard in order to maximize portability, minimize excessive memory usage and code complexity. Flek avoids all of the following:

  1. Multiple inheritance
  2. Runtime type identification
  3. Templates
  4. The Standard Template Library
  5. Exceptions
  6. Namespaces
  7. New style casts

As an application developer, you may choose to use any or all of the above standard c++ features in your application. And if you don't use any of the above features Flek won't force them on you. Flek intends to be usable by as many programmers, and in as many programming environments as possible.
